# OmniLSS 六个月路线图开发计划（2026-05-18）

> English version: [six-month-roadmap-development-plan-2026-05-18.md](six-month-roadmap-development-plan-2026-05-18.md)
>
> 来源路线图：面向 0.3.0-dev 的 OmniLSS 六个月路线图，推进顺序为 **技术可用性 → 完整性 → 准确性 → 商业化**。

## 开发语言与文档策略

- 代码、注释、提交信息、Issue 标题和内部开发协作默认使用英文。
- 面向用户和贡献者的规划文档同时维护英文和中文版本。
- 中文文档在英文文档同名基础上，在 `.md` 前增加 `_cn`。
- 每一组中英文文档都必须在文档顶部互相引用。

## 优先级模型

| 优先级 | 含义 | 发布门禁 |
|---|---|---|
| P0 | 可能立即造成用户损失的问题，包括错误结果或崩溃。 | 必须在任何扩展工作前修复。 |
| P1 | 宣称存在但不能稳定工作的功能。 | 必须在公开可信度建设前修复。 |
| P2 | 可以运行但准确性或完整性不足的功能。 | 必须在大范围学术验证前处理。 |
| P3 | 提升学术或商业价值的完整性扩展。 | 在正确性基础稳定后推进。 |
| P4 | 商业化基础设施。 | 在核心可靠性和验证门禁通过后推进。 |

## 工作流拆解

### 第一阶段，第 1-2 周：紧急稳定基线

**目标：** 消除 P0 缺陷，并修复最高风险的 P1 可信度问题。

1. **修正含平滑项模型的 df_fit/AIC/SBC。**
   - 在独立 RS 和 CG 后端的模型组装阶段，用有效自由度（EDF）替代平滑基函数的名义列数。
   - 验收：有惩罚平滑项时，模型自由度低于未惩罚基函数列数，AIC/SBC 使用修正后的自由度。
2. **在 CG 跨导数后端中使用分布感知残差。**
   - 复用通用的随机分位数残差构造器和回退残差函数。
   - 验收：非 Normal 分布不再从 CG 返回硬编码的 `y - mu` 原始残差。
3. **明确 gRPC 服务依赖与 stub 生成流程。**
   - 将 `grpc` 可选依赖统一到 protobuf v4+，并保留生成工具依赖。
   - 增加可重复执行的 protobuf 生成命令。
   - 验收：贡献者无需猜测命令即可重新生成 stubs。
4. **限制 gRPC 模型注册表增长。**
   - 为模型文件增加 TTL 和最大条目淘汰策略。
   - 验收：超过容量时淘汰旧模型，过期模型会被删除。
5. **在规划中澄清历史 L-BFGS 后端命名。**
   - 保持向后兼容，但明确它不是标准 Cole-Green 算法。

### 第二阶段，第 3-4 周：标准 Cole-Green 实现

**目标：** 提供与文献一致、可对照 R `gamlss(method="CG")` 验证的 eta 尺度 Cole-Green 实现。

1. 新增基于 eta 尺度得分与 Hessian 跨导数修正的 `cole_green.py` 后端。
2. 默认将 `gamlss(method="CG")` 路由到标准 Cole-Green 后端，同时保留显式实验后端名称用于对比研究。
3. 在 R 可用时，增加 NO、GA、NBI、BCCG 和 TF 的一致性测试。
4. 增加 CG vs RS 的收敛性和 deviance 对比基准报告。

### 第三阶段，第 5-6 周：数值验证与测试基础设施

**目标：** 让数值可信度可重复验证。

1. 增加 R vs Python 一致性测试 CI。
2. 将平滑参数更新延后到系数经历若干预热迭代后再开始。
3. 引入分布能力注册表，记录每个分布对算法、离散性、零膨胀和导数能力的支持情况。

### 第四阶段，第 7-10 周：完整性与鲁棒性

**目标：** 移除主路径中的未完成功能，并提升算法路由可靠性。

1. 实现基于能力的 Mixed 算法自动路由。
2. 实现 BCa bootstrap 置信区间。
3. 接入 TPS/B-spline REML 平滑参数选择。
4. 支持更高阶 B-spline 导数。

### 第五阶段，第 11-16 周：Deep GAMLSS 与学术准备

**目标：** 强化 OmniLSS 相对 R `gamlss` 的差异化能力。

1. 增加部署级 Deep GAMLSS 预测 API、分位数预测、CRPS 支持与交叉验证。
2. 增加半参数 Deep GAMLSS，将可解释线性项与神经网络非线性项结合。
3. 准备并维护 JOSS 论文、案例、基准和一致性证据。
4. 将生产相关序列化路径升级为安全的 JSON 工件。

### 第六阶段，第 17-26 周：商业化基础设施

**目标：** 准备生产友好的 API 和发布条件。

1. 完成生产级 gRPC 传输，包括流式进度和 Arrow 负载。
2. 升级模型存储，支持有界本地持久化和可选 Redis 持久化。
3. 实现 Pro 层 AutoGAMLSS 工作流和批量预测 API。
4. 在代表性设备上校准 GPU 切换阈值。
5. 完成 v0.3.0 的 PyPI 发布检查。
