# OmniLSS v0.3.0 发布前阶段性收尾记录（2026-05-17）

[中文版本](v0.3.0-release-closure-2026-05-17_cn.md)

本文是 v0.3.0 发布前的阶段性收尾清单。目标不是继续扩张功能范围，而是把已经完成的稳定化工作、发布门禁、环境限制和后续遗留项固定下来，避免在发布前继续引入新的不确定性。

## 结论

**v0.3.0 可以作为稳定化候选版本收尾。** 当前版本号已经统一为 `0.3.0`，CHANGELOG 已收敛到本版本的真实变更范围，README 已补充发布与验证口径，CG 算法的“名实不符”风险已经拆分为：

1. v0.3.0 发布说明中如实描述当前可验证实现；
2. 后续开发文档中明确交叉导数完整实现路径，不把未验证的算法收益包装成已完成能力。

## 本轮收尾范围

### 已冻结事项

- 不在 v0.3.0 发布前继续新增分布、优化器、公式语法或平滑方法。
- 不把 Python-only smoke check 解释为 R 等价性证明。
- 不把未完成的 CG 性能与收敛优势写成发布承诺。
- 不把打包工具缺失视为代码失败；它是当前容器的环境限制，但仍是正式发布前必须补跑的门禁。

### 已完成事项

- 版本号收敛到 `0.3.0`。
- README 中的性能与验证声明已转为门禁化、条件化表述。
- `CHANGELOG.md` 的 v0.3.0 条目已补充验证状态和已知限制。
- v0.3.0 稳定化进度文档已从“开发清单”更新为“发布候选收尾状态”。
- CG 交叉导数后续方案已形成独立开发路径文档。

## v0.3.0 发布说明建议

推荐发布标题：

> OmniLSS v0.3.0 — stabilization release, transparent validation gates, and CG cross-derivative roadmap

推荐发布正文要点：

- 稳定化版本：重点是架构边界、发布口径、注册表、文档与测试门禁，而不是大规模新增功能。
- 性能声明透明化：必须注明硬件、JAX backend、JIT warm-up 与 R 可用性。
- CG 状态透明化：当前已有 full observed-information 路径与 CG v2 外循环探索；后续将把 eta-level cross-derivative correction 与 coefficient-level Hessian 路径统一到同一验证门禁。
- 环境限制透明化：如发布环境没有 R 或打包依赖，必须把相关验证标注为未执行，而不是发布为“已通过”。

## 后续不阻塞 v0.3.0 的遗留项

这些事项进入 v0.3.x 或 v0.4.0，不再阻塞 v0.3.0 发布收尾：

1. 将 `fitting.py` 的辅助函数完全迁移到 `_fitting_*` 模块并移除所有重复定义。
2. 在 CI 中安装完整 dev 依赖后恢复架构 smoke test 的强制门禁。
3. 在具备 R 的环境中复跑 R-backed consistency gate。
4. 根据 `docs/development/cg-cross-derivatives-development-path-2026-05-17.md` 推进 CG 交叉导数实现统一。
5. 对 CG / RS / Mixed 的对比结果生成可引用 benchmark artifact。
